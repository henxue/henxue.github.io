这样基本就差不多了，接下来就要整一下分页符和上传图片的。

百度编辑器的分页符是pagebreak，先在ueditor.all.js文件里面搜索一下，大概在11152行，原始代码如下：

/**
* 分页功能插件
* @file
* @since 1.2.6.1
*/
UE.plugins['pagebreak'] = function () {
var me = this,
notBreakTags = ['td'];
me.setOpt(‘pageBreakTag’,'_ueditor_page_break_tag_’);

function fillNode(node){
if(domUtils.isEmptyBlock(node)){
var firstChild = node.firstChild,tmpNode;

while(firstChild && firstChild.nodeType == 1 && domUtils.isEmptyBlock(firstChild)){
tmpNode = firstChild;
firstChild = firstChild.firstChild;
}
!tmpNode && (tmpNode = node);
domUtils.fillNode(me.document,tmpNode);
}
}

我们需要做的就是把_ueditor_page_break_tag_改为帝国的分页符就可以了。

搞定分页符，就来搞定一下图片路径。

一开始我并没有修改，所以图片一直上传不了。在网上找了一下教程，发现都是老版本的，和新版本的根本对不上。老版本的是在ueditorphpimageUp.php中修改，但是新版本却是在同目录下的config.php文件中修改，默认代码如下：

<?php

return array(

//图片上传允许的存储目录
‘imageSavePath’ => array (
‘upload1′, ‘upload2′, ‘upload3′
)

);

也就是说，默认的情况下，可以有三个目录让你选择，然后文件是上传在ueditorphpupload1 下面。但是奇怪的是，我新建了这个文件夹，也是传不上，就没深入去研究了。于是就把upload1改成../../d/files/upload，这个路径是存在的。再试一下上传，还真可以了，那就这么用着吧。

帝国CMS整合百度编辑器到这里也算是结束了，由于技术和时间有限、在基本能达到我的需求的情况下，就这么用着吧。同样还是有一个问题，上传图片没有水印，但是在我误打误撞的情况下，竟然可以轻松实现了水印，而且图片也能按照帝国系统设置的存在d/file下（和上面设置的路径没关系，听我道来）。

刚我上面说了，我习惯把这几个都点上

在用百度编辑器上传完图片、写完文章点击提交以后，图片变成这个样子了/d/file/2013/12/20/e7ecf7fc944a0bca0ea29cf0c550eaa0.jpg”，也就是说，和我们平时用默认的编辑器上传是一样的。第二次我试了一下把远程保存图片给去掉勾，就真的传到我设置的../../d/file/upload目录下了。

那原因就是这样：

图片先被百度编辑器上传到了../../d/file/upload目录下，在我们提交文章以后，帝国的远程保存图片功能开始工作了，就把http://www.xx.com/d/file/2013/12/20/e7ecf7fc944a0bca0ea29cf0c550eaa0.jpg这个文件给当成远程文件保存到了我帝国程序设置的/d/file/2013/12/20/目录下了。也就是说，在服务器上，是存在了两个同样的文件。那么如果为了节省空间，可以定期处理../../d/file/upload下的文件了。当然，如果你没有远程保存图片，而是继续使用你在编辑器设置的路径的话，也是可以的。

如果大神们有更好的办法，麻烦告知一下，谢谢！
