帝国CMS中$navinfor和$GLOBALS的一些使用小技巧

  • henxue 发布于 4小时前
  • 分类：帝国教程
  • 阅读()
  • 评论

$navinfor和$GLOBALS是两个在模板制作当中比较常用的函数，这两个函数有啥作用呢？功能到底有多强大？这个就要看模板制作的需求和配合其他功能的使用了。这里列举几个比较常用的方法。点击这里[1]、[2]可以查看官方对这两个函数的简单解释。

下面是我根据帝国官方提供的教程而实现的几种效果：

在内容模板中调用出filename字段

前提：由于我在下载系统模型下的文章都是自己定义文件名的，也就是filename，然后在自定义列表中有用到该filename，且必须在下载系统模型的内容也中调出filename字段。可以使用<?=$navinfor[filename]?>即可。当然，你也可以调其他字段，比如<?=$navinfor[keyboard]?>。

在列表中利用$GLOBALS给当前专题的导航添样式

代码如下：

<a href=”[!--news.url--]android” title=”安卓游戏” <? if ($GLOBALS[navclassid]==1)
{
echo “class=’on’”;
}?>>安卓游戏</a> <a href=”[!--news.url--]ios” title=”iOS游戏” <? if ($GLOBALS[navclassid]==2)
{
echo “class=’on’”;
}?>>iOS游戏</a>

这是我某个网站中的两个专题，使用的是同一个列表模板。如果按照传统的做法，就得做两个列表模板，在安卓游戏的列表模板中给安卓游戏加上on，然后在iOS列表模板中给iOS加上on，这样是很浪费的。那么我们可以利用专题的ID和$GLOBALS来实现图片的效果。思路是这样的：安卓游戏的专题ID是1，iOS的专题ID是2.那么在安卓游戏这边判断当前专题ID是否等于1，如果等于1就加class=&quot;on&quot;样式。iOS地方也是同样的道理。这样就很简单的实现了。更多的功能，还需要自己的挖掘。

在列表页、内容页中判断在某栏目下、某标签的样式为none

用法就是，某个标签（以<li>为例）的样式，在栏目ID为1下显示，其他栏目下不显示。

在</head>前面加一个判断（列表页为例）：

<?php
if($GLOBALS[navclassid]==1)
{
?>
<style>.ceshi{}</style>
<?php
}
else
{
?>
<style>.ceshi{display:none}</style>
<?php
}
?>

这样一来就达到了以上想要的效果
